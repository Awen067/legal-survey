# 企业用工法律风险体检小程序 - 数据库版升级说明

## 📊 升级概述

本次升级将系统从**文件系统存储**改为**SQLite数据库存储**，主要优势：

### ✅ 升级优势

1. **数据持久化**：服务器重启后数据不丢失
2. **多用户共享**：管理员可从任何设备查看所有提交记录
3. **数据完整性**：支持事务处理，避免数据损坏
4. **备份恢复**：提供数据库备份和恢复功能
5. **性能优化**：数据库索引加速查询

### 📁 文件变化

| 文件 | 说明 |
|------|------|
| `server.py` | 原版服务器（文件系统存储） |
| `server_db.py` | **新版服务器（SQLite数据库存储）** |
| `server_backup.py` | 原版服务器备份 |
| `start.sh` | 启动脚本（支持选择版本） |
| `surveys.db` | SQLite数据库文件（自动创建） |

## 💾 数据库说明

### 数据库文件位置
- **路径**：`/Users/wangwen1/Desktop/0000/体检/survey_app/surveys.db`
- **类型**：SQLite 3数据库
- **大小**：随数据量增长（每1000条记录约10MB）

### 数据库表结构

```sql
CREATE TABLE surveys (
    id TEXT PRIMARY KEY,           -- 提交ID（8位UUID）
    company_name TEXT,              -- 企业名称
    contact TEXT,                  -- 联系方式
    answers TEXT,                  -- 答案JSON
    risk_items TEXT,               -- 风险分析结果JSON
    docx_path TEXT,                -- DOCX报告文件路径
    report_path TEXT,              -- HTML报告文件路径
    docx_id TEXT,                  -- DOCX文件ID
    report_id TEXT,                -- HTML报告ID
    created_at TEXT,               -- 创建时间（YYYY-MM-DD HH:MM:SS）
    status TEXT DEFAULT 'pending'   -- 状态（pending/processed）
);

CREATE INDEX idx_surveys_created_at 
ON surveys(created_at DESC);  -- 加速按时间查询
```

## 🔍 数据库查询示例

### 使用SQLite命令行工具

```bash
# 连接到数据库
sqlite3 surveys.db

# 查询所有提交记录
SELECT id, company_name, contact, created_at FROM surveys ORDER BY created_at DESC;

# 查询今日提交
SELECT COUNT(*) FROM surveys WHERE created_at LIKE '2026-06-10%';

# 查询风险项最多的前10家企业
SELECT company_name, LENGTH(risk_items) - LENGTH(REPLACE(risk_items, '}', '')) as risk_count 
FROM surveys 
ORDER BY risk_count DESC 
LIMIT 10;

# 导出所有提交记录为CSV
.mode csv
.headers on
.output submissions.csv
SELECT * FROM surveys;
.quit
```

### 使用Python脚本

```python
import sqlite3
import pandas as pd

# 连接到数据库
conn = sqlite3.connect('surveys.db')

# 读取所有数据
df = pd.read_sql_query("SELECT * FROM surveys", conn)

# 数据分析
print(f"总提交数：{len(df)}")
print(f"平均风险项：{df['risk_items'].apply(lambda x: len(eval(x)) if x else 0).mean():.1f}")

# 关闭连接
conn.close()
```

## ⚠️ 注意事项

### 1. 数据库文件权限
确保`surveys.db`文件有读写权限：
```bash
chmod 644 surveys.db
```

### 2. 数据库锁定
如果服务器异常退出，可能导致数据库锁定。解决方法：
```bash
# 检查是否有进程占用
lsof | grep surveys.db

# 杀死占用进程
kill -9 <PID>
```

### 3. 数据库损坏恢复
如果`surveys.db`损坏，可以使用备份恢复：
```bash
# 从备份恢复
cp surveys.db.bak surveys.db

# 或使用SQLite修复工具
sqlite3 surveys.db ".recover" | sqlite3 surveys_new.db
mv surveys_new.db surveys.db
```

### 4. 多服务器部署
如果要在多台服务器上部署，需要：
- **共享数据库**：将`surveys.db`放在共享存储（如NFS、SMB）
- **或主从复制**：使用SQLite的`rsync`定期同步

**推荐方案**：使用单个服务器 + 数据库备份

## 🐛 故障排查

### 问题1：服务器启动失败

**错误信息**：`sqlite3.OperationalError: unable to open database file`

**解决方法**：
```bash
# 检查目录权限
ls -la /Users/wangwen1/Desktop/0000/体检/survey_app/

# 创建数据库目录（如果不存在）
mkdir -p /Users/wangwen1/Desktop/0000/体检/survey_app/

# 设置权限
chmod 755 /Users/wangwen1/Desktop/0000/体检/survey_app/
```

### 问题2：提交数据未保存到数据库

**检查方法**：
```bash
# 查看数据库内容
sqlite3 surveys.db "SELECT COUNT(*) FROM surveys;"

# 查看最新提交
sqlite3 surveys.db "SELECT * FROM surveys ORDER BY created_at DESC LIMIT 1;"
```

**可能原因**：
- 数据库文件权限不足
- 磁盘空间不足
- SQLite版本过旧

### 问题3：管理员后台无法显示数据

**检查方法**：
1. 打开浏览器开发者工具（F12）
2. 查看Console标签页是否有错误
3. 查看Network标签页，`/api/submissions`接口是否返回数据

**解决方法**：
- 清除浏览器缓存
- 重启服务器
- 检查数据库文件是否完整
